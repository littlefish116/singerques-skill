# 贡献指南（Contributing）

感谢你愿意让这个知识库更准、更新！本库的目标是帮大众听审报名者**把真实的自己讲清楚、讲自然**——所以贡献也围绕"**真实、脱敏、可溯源**"三件事。

> 投稿前请先读 `SKILL.md` 第 5 节**禁止事项**与附的**核心立场**：本库**不收录**任何"编造经历 / 规避审查 / 伪装身份 / 隐藏社交账号"的内容。

---

## 一、能贡献什么

- 新的**问卷题 / 电话面试题 / 追问 / 陷阱题**
- 新的**高风险表述 / 雷区**
- **当季规则、流程、阵容**变化（标明来源与年份）
- **入选 / 落选 / 被质疑**的真实经历（须脱敏）
- 对已有内容的**勘误**

> ❌ 不收：纯八卦 / 赛果讨论 / 情绪宣泄；任何教你作弊 / 伪装的内容；含未脱敏隐私的投稿。

## 二、怎么投（GitHub 流程，推荐）

1. **开 Issue**：用脱敏后的内容开一个 Issue，标题写 `[类别] 简述`，正文给：**类别 / 内容 / 来源类型**（自己经历 / 听说的 / 推测）。
2. **脱敏**：投稿前**必须**脱敏——电话 / 邮箱 / wxid / 身份证 / 真实姓名 / 社交链接 / @昵称 / 群名。可本地跑 `scripts/submit.py` 自动脱敏后，再把结果贴进 Issue。
3. **等复核**：维护者按"待审 → 交叉印证 → 定来源强度"评审；通过后以 PR 并入对应 `references/` 文件，并在 `CHANGELOG.md` 留痕。

> 也可走本地 `submissions/` 接口（`submit.py` / `_template.md` → `_inbox.md`），适合不开 Issue 的离线贡献；流程与上面一致。

## 三、来源强度（投稿必标，合并时复核）

| 标签 | 含义 |
|------|------|
| `高` | 多人群、跨期反复印证（**新投稿默认不给高**，需评审交叉印证后才能升） |
| `中` | 个别一手陈述、内部自洽 |
| `低` | 孤立、单一来源、未佐证 |
| `推测` | 猜测、未经证实 |
| `待证` | 新投稿的默认状态 |

合并时一律标 `高/中/低/推测`，与现有 references 口径一致；**不自动覆盖**已有的 `高` 结论。

## 四、守门红线（以下拒收，不进知识库）

- ❌ "如何**编造经历 / 规避审查 / 伪装身份 / 隐藏社交账号**"——违反 `SKILL.md` 禁止事项。
- ❌ 含**未脱敏隐私**（电话 / 邮箱 / wxid / 身份证 / 真实姓名 / 社交链接 / 群昵称）。
- ❌ 与报名 / 面试表达无关的纯八卦、赛果、情绪宣泄。

## 五、跨年以近年为准

节目赛制 / 阵容 / 题型逐年演进。跨年信息冲突时，**以近年为准（2026 > 2025 > 2024）**；旧经验标注年份，不覆盖新结论。

## 六、隐私（重要）

- 公开库**只发布 `references/` 里的脱敏结论**；**原始群聊、清洗语料不入库**（见 `.gitignore`）。
- 投稿**不要附带他人的真实身份信息**；亲历经历请脱敏到无法识别个人。
- 本 skill 会在使用者本地（已 gitignore 的 `_local/`）存储其真实情况——那是**使用者本地数据，不属于本库**，请勿在投稿里附带。
