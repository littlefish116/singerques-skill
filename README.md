# 《歌手2026》大众听审准备 Skill

帮你准备大众听审的**报名问卷、电话面试、现场表达与自我介绍**——基于你**真实**的情况组织回答，让它真实、自然、理性、不饭圈化、可信。

> ⚠️ 它**不会**帮你：编造经历、规避审查、伪装身份、隐藏社交账号、或给可背诵的"标准答案"。它只帮你说清楚**真实的你自己**。

---

## 🔒 隐私 & 这个仓库里有什么

这是个**知识库 skill**。为保护隐私，仓库严格分公开 / 私有两层：

| 公开（随仓库发布） | 私有（`.gitignore`，永不提交） |
|---|---|
| `SKILL.md` / `references/`（全部脱敏、标来源强度的结论） | `*@chatroom.jsonl`（原始群聊导出） |
| `scripts/`（脱敏 / 提交工具） | `cleaned_records/`（清洗语料，源自私有群） |
| `submissions/` 模板与说明 | `submissions/_inbox.md`（维护者待审工作区） |
| `CONTRIBUTING.md` / `CHANGELOG.md` / `LICENSE` | `_local/`、`my_profile.md`、`my_drafts/`（**你的**个人数据） |

> **你的个人数据不会进仓库**：本 skill 若在本地存你的真实情况（听歌偏好 / 动机 / 年龄 / 职业等），只写在已 gitignore 的 `_local/`，不离开你的机器。详见 `CONTRIBUTING.md` 第六节。

## 一、快速上手（3 步）

**1. 装成 Claude skill**

入口是本目录的 `SKILL.md`。把整个目录放到 skills 路径下即可（目录名随意，skill 名取自 `SKILL.md` 的 `name` 字段，默认 `singerques-skill`）：
- 个人范围：`~/.claude/skills/singerques-skill/`
- 或项目范围：`<你的项目>/.claude/skills/singerques-skill/`

**2. 调用它**

- **斜杠命令**：`/singerques-skill`（不带参数会列出子指令目录）
- **子指令直达**：`/singerques-skill <名称或缩写>`，或自然语言（"跑漏斗""审一下这段""投喂个新题"）

**3. 准备好"真实底牌"**

它会先问你几个真实情况（见第三节）。提前想好：真实听什么歌、看过什么现场、为什么想去、有没有偏爱某位在线歌手、年龄段与职业。

### 子指令目录（带缩写）

| 子指令 | 缩写 | 干什么 |
|---|---|---|
| 模拟筛选（漏斗） | `sim` | 跑完整报名→入选漏斗，逐关判定 🟢/🟡/🔴 + 给改法 |
| 补充信息（投喂） | `feed` | 投喂新问卷题/雷区/规则/经历，走 待审→守门→留痕 |
| 疑问咨询 | `ask` | 伞形入口，下挂：问卷填写 `form` / 面试准备 `iv` / 风险自检 `risk` / 现场表达 `live` / 流程机制 `flow` |

## 二、它能帮我做什么（附示例 prompt）

### 模拟筛选（漏斗）
> `/singerques-skill sim` —— "用我的真实情况，从问卷关走到入选关，逐关告诉我能不能往下走、卡在哪、怎么改。"

### 填报名问卷
> `/singerques-skill form` —— "我在填《歌手2026》报名表，'喜欢的曲风'和'观演动机'怎么写？我听流行和民谣，看过一场音乐节。"

### 准备 / 模拟电话面试
> `/singerques-skill iv` —— "模拟一下三轮电话，你当面试官出题，我答，你给我的回答挑刺。"

### 判断某段回答有没有风险
> `/singerques-skill risk` —— "我打算这么答：'我特别喜欢XX，他唱功无敌，我一定投他。'——有风险吗？帮我改自然。"

**你通常会收到这样的回复结构**（组织一段回答时）：
```
推荐回答  — 用你的真实情况填的方向（不是套话）
回答思路  — 为什么这样答、对方考察什么
风险点    — 可能被读成粉圈化/矛盾/背答案的地方
可能追问  — 对方接下来可能问什么
更自然版本 — 改成"像跟朋友聊音乐"的口语版
```

## 三、它会先问你什么（第 0 步 · 真实情况采集）

为了让回答**基于你的真实情况**（而不是编造），它通常先问：

1. 平时真实听什么歌？喜欢的曲风 + 两三个类型不同的歌手（各说一句"为什么"）。
2. 看过哪些现场 / 音综录制？（没有也完全没问题）
3. 为什么想去《歌手2026》当大众听审？（真实动机）
4. 本季在线歌手里，有没有你特别喜欢 / 在追的？（**如实**说——用于诚实应对，不是帮你隐藏）
5. （可选）看一场演出，最看重什么？
6. 你的**年龄段**（10+/20+/30+/40+/50+）与**职业**？（年龄用于按节目组五年龄组配额校准入选概率；职业用于判断是否走音乐专业深挖路径。都如实说，不评判好坏）

> 这几条是**你真实情况**的提炼，不是人设模板。如实回答即可；不会的就直说"没有/没关注"。

## 四、可信度怎么看 & 局限

**可信度图例**（知识库里每条结论都标了）：
- `高` 多人群、跨期反复印证
- `中` 个别过来人的具体一手陈述
- `低` 孤立、单一来源、未佐证
- `推测` 群友猜测、未经证实

**局限**：
- 知识库来自群聊**文本**讨论；问卷/规则的**截图、语音**当时无法解码，是已知缺口（可后续补 OCR）。
- 具体题目多为个人复述，**题型套路**可信度高，**当季具体歌手/曲目**要你自己换成最新的。
- 筛选含**随机成分**，**没有任何回答能保证入选**——它只帮你更真实可信地表达。
