# 回答风险自检表（risk_selfcheck）

> 用法：把用户起草的（或你给出的）一整段回答，按下面 **5 个维度**逐条对照。命中任一"红旗"就记一笔，最后看总数。
> 与 `risk_phrases.md` 配合：risk_phrases 讲"哪些词/行为是雷"，本表讲"怎么给一整段回答打分"。
> 可信度标注同其他 references：`高` / `中` / `低` / `推测`。

---

## A. 粉圈化信号（命中即高风险）

- [ ] 明确表态"喜欢 / 支持 / 一定投"某位**在线歌手**
- [ ] 用饭圈黑话：控评 / 打投 / 数据 / 超话 / 应援 / 塌房 / 唯粉 / 拉踩 / CP…
- [ ] 贬低某歌手来抬高另一位
- [ ] 情绪过载：激动、护短、替某歌手鸣不平
- [ ] 粉圈称呼："我们家 / 哥哥 / 姐姐"

> 真实后果印证：表露对某首发歌手的喜欢 → 对方"立马挂电话"。`高`

## B. 功利化 / 目的性过强

- [ ] 通篇在"怎么被选上 / 怎么过关"，而非真实表达
- [ ] "我一定能入选 / 一定能过"类承诺（也是 SKILL.md 禁止的虚假承诺）
- [ ] 把去现场说成功利目的（追星 / 见偶像 / 刷履历）

> 修正：动机落回"想看一场好演出 / 亲历一次顶级现场"。

## C. 背答案 / 模板感

- [ ] 句式工整、像作文，无口语停顿
- [ ] 网络模板句（"始于颜值陷于才华""声音里有故事"…）
- [ ] 秒答 + 长篇，没有"嗯…我想想""这个我不太确定"
- [ ] 细节经不起追问（一说"哪期 / 哪个现场"就卡壳）
- [ ] AI 味（"作为一个音乐爱好者，我认为…"的套路开头）

> 真实加分项反证：入选者"一个字一个字写出来的一点没用 AI"。`中`

## D. 前后矛盾

- [ ] 与报名表内容对不上（报名表写 A，口述说 B）
- [ ] 三轮电话之间口径不一
- [ ] 自称"听歌杂"却只说得出在线歌手
- [ ] 评分标准前后不一（一会看唱功、一会看感情）
- [ ] 嘴上说没关注某歌手，细节却很熟

> 修正：以**真实**为单一信息源——真话不用记，编的才需要记。

## E. 不可信 / 经不起追问

- [ ] 编造没去过的现场、没听过的歌手
- [ ] 评价空泛（只有"好听 / 厉害"，说不出为什么）
- [ ] 关键细节模糊或前后对不上
- [ ] 过度包装身份（假装音乐专业 / 假装某类听众 / 伪造乐迷履历）

---

## 打分与处置

- **0 红旗** → 🟢 安全，可用（仍建议走一遍"更自然版本"润色）。
- **1–2 红旗**（且未命中 A 类致命项）→ 🟡 需调整：按各条"修正"改。
- **≥3 红旗**，或**命中 A 类任一致命项**（明确站队 / 饭圈黑话 / 贬低他人 / "一定投某歌手"）→ 🔴 重写。

## 如何得到"更自然版本"

1. 把书面腔改口语：拆短句，加"就…吧""我个人感觉""说不上来为什么，但…"。
2. 删掉所有饭圈黑话与绝对站队，落点改回"看现场综合表现"。
3. 用一个**真实**细节替换每个空泛形容词（"唱功好" → "副歌那个高音他稳住了"）。
4. 允许不完美：加一处"这个我没太关注""见仁见智"。
5. 念一遍——像不像在跟懂音乐的朋友随口聊；像作文就再改。
